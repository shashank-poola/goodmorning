# Good Morning

> One command center to replace the 20-tab morning ritual.

A personal dashboard for founders and independent consultants. Pulls in your Google Calendar, Gmail, LinkedIn, Twitter/X, GitHub trending, and news — with an LLM intelligence layer on top — all without handing credentials to any third party.

Built with React 18 + TypeScript + Vite (frontend) and Hono + Node (backend). ~106 tests, strict mode throughout.

---

## What it does

| Widget | Data source | Intelligence |
|---|---|---|
| **Meetings** | Google Calendar OAuth | Free-time gaps between events |
| **Gmail** | Gmail OAuth (primary/important only) | "Why it matters" one-liner per email |
| **Important Emails** | Gmail OAuth | Filtered + LLM-ranked |
| **Tweets** | RSS (Nitter / RSSHub mirrors) | Per-tweet insight via GLM |
| **LinkedIn** | Manual paste | Reply drafts generated by LLM |
| **News & GitHub** | RSS feeds + GitHub Trending API | Per-headline summary via GLM |
| **To-Do** | File-backed store (server) | Auto-priority + duplicate detection |
| **Yesterday Recap** | LLM synthesis across all widgets | Morning briefing bullets |
| **Stocks** | Live ticker (TickerBar) | — |
| **Claude Usage** | Usage API | Daily token/cost tracking |
| **Finance / Renewals** | Local config | Overdue alerts surfaced in top bar |
| **Reminders** | Local config | — |

The compose bar at the bottom lets you draft a post once and toggle LinkedIn / X / Substack / Newsletter as targets.

## Architecture

```
src/
  data/            # DataProvider interface, MockDataProvider, ApiDataProvider, types
  components/      # Panel, WidgetBody, useWidgetData hook
  shell/           # TopBar, Sidebar, DashboardGrid, TickerBar, ComposeBar, CommandPalette
  widgets/<Name>/  # one folder per widget — component + module.css + test
  styles/          # tokens.css (design system), global.css
  hooks/           # useAuth, useKeyboardShortcuts, ThemeContext, AuthContext

server/src/
  services/        # googleCalendar, googleGmail, rssNews, tweetRss, githubTrending,
                   # todoService, intelligenceService, glmClient, googleAccounts
  data/            # newsFeeds, tweetFollows (file-driven follow lists)
  app.ts           # all API routes (Hono)
  index.ts         # server entry
```

**The key design:** every widget reads from `src/data/DataProvider.ts` — a single typed interface. `providerFactory.ts` is the swap point between `MockDataProvider` (zero setup, realistic fake data) and `ApiDataProvider` (real backend). Adding a new data source never touches a widget.

## Quick start

### Frontend only (mock data, no backend needed)

```bash
npm install
npm run dev          # → http://localhost:5173
```

### Full stack (real Google data + LLM intelligence)

**Terminal 1 — Backend**
```bash
cd server
cp .env.example .env   # fill in credentials (see below)
npm install
npm run dev            # → http://localhost:3001
```

**Terminal 2 — Frontend**
```bash
cp .env.example .env   # set VITE_USE_API=true
npm run dev            # → http://localhost:5173
```

Then open **http://localhost:5173** and click your avatar → **Connect Google** to run the OAuth flow. Add a second Google account anytime from the same menu.

## Environment variables

### `server/.env`

```env
# Google OAuth (create at console.cloud.google.com)
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
GOOGLE_REDIRECT_URI=http://localhost:3001/auth/google/callback

# Zhipu AI (glm-5.2) — for email insights, news summaries, tweet insights,
# LinkedIn reply drafts, recap, todo categorization
ZAI_API_KEY=
ZAI_MODEL=glm-5.2          # optional, this is the default

# After OAuth, browser is redirected here
FRONTEND_URL=http://localhost:5173

# Your local timezone (IANA format) for calendar windowing
CALENDAR_TIMEZONE=Asia/Kolkata

PORT=3001
```

### `.env` (frontend root)

```env
VITE_API_BASE_URL=http://localhost:3001
VITE_USE_API=true
```

## Privacy constraints (by design)

- **No credential sharing.** Only Google OAuth tokens (stored locally in `server/.token-store.json`) and the user's own ZAI API key. No passwords, session cookies, or third-party automation tokens.
- **Twitter/X via RSS only.** No X API key required — uses public Nitter / RSSHub mirrors with error-content filtering.
- **LinkedIn via manual paste.** The user copies their post/comments, the LLM generates reply drafts for manual copy-paste back. Zero LinkedIn credentials.
- **LLM calls are additive.** Every widget degrades gracefully if `ZAI_API_KEY` is absent — data still shows, just without the insight layer.

## Adding a widget

1. Add the data type to `src/data/types.ts` and a method to `src/data/DataProvider.ts`
2. Add mock data to `src/data/MockDataProvider.ts`
3. Add a real fetch to `src/data/ApiDataProvider.ts`
4. Create `src/widgets/<Name>/` — component, `module.css`, and a test
5. Add a grid area in `DashboardGrid.module.css` (all three breakpoints) and render it in `DashboardGrid.tsx`
